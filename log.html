1. Executive Summary

OriginAnecdote is a premium, client-side single-page dashboard application built specifically for educators conducting classes for international students. The project tackles timezone differences and calendar visibility issues associated with global tutoring, offering booking request verification, automated local student clock checks, dynamic timezone simulation, and visual student analytics.

2. Research & Requirements Phase

Before starting development, research was conducted on leading online tutor dashboards such as Preply and italki. These platforms provide teachers with:

  • Automatic Timezone Shifting: Translating slots automatically based on student and teacher account regions.
  • Recurring Weekly Availabilities: Allowing teachers to open set hours to structure their weeks.
  • Safety Notice Policies: Preventing last-minute bookings (e.g. requiring 12-24 hours notice) to avoid scheduling conflicts.

OriginAnecdote addresses these areas with specific frontend modules:

  1. An Interactive Timezone Simulator allowing the teacher to view their calendar from the perspective of students in Tokyo, São Paulo, Paris, London, New York, or Beijing.
  2. An Alert Helper indicating whether a proposed student request lands in a reasonable study hour (daytime) or early morning/late night for the student.
  3. Live Student Clocks displaying real-time local times on CRM cards, complete with visual alerts indicating when students are likely sleeping.

3. System Architecture & Tech Stack

To ensure instantaneous load times and simple deployment, the system is designed as a standalone static web application:

  • HTML5 structure: Semantic markup dividing the viewport into a sticky navigation sidebar and dynamic content views.
  • Vanilla CSS (Responsive Layouts): A global design system using HSL color variables. Uses a dark glassmorphism styling approach, containing linear gradients, backdrop-filters, custom scrollbars, and micro-animations.
  • Vanilla JavaScript (ES6 Modules): Client-side model-view-controller (MVC) logic. Manages states for calendar availability, timezone calculation offsets, notifications, student search queries, and modal rescheduling dialogs.

4.3 JavaScript State Controller (app.js)

Coordinates time translations, updates clocks every second, and updates availability grids dynamically. Here is the timezone translation helper: 

function translateTime(hour, sourceTz, targetTz) {
  const sourceOffset = TZ_OFFSETS[sourceTz] || 0;
  const targetOffset = TZ_OFFSETS[targetTz] || 0;
  const diff = targetOffset - sourceOffset;
  
  let translatedHour = hour + diff;
  let dayOffset = 0;

  if (translatedHour >= 24) {
    translatedHour -= 24;
    dayOffset = 1;
  } else if (translatedHour < 0) {
    translatedHour += 24;
    dayOffset = -1;
  }
  return { hour: translatedHour, dayOffset: dayOffset };
}
